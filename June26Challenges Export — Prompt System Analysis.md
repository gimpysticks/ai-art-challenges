The ZIP export shows that your Midjourney workflow is not simply a list of prompts. It is a **modular prompt operating system**. The system has one large master identity framework and several challenge-specific rule packs that override or specialize the master rules depending on the collaboration or prompt list.

## Most important operational rule

The uploaded files show that you already think in structured systems, but your Instagram performance issue comes from turning too many daily outputs into feed posts. The template system should now be used as a **selection engine**, not just a generation engine.

A daily prompt should pass through this filter:

| Question | If yes | If no |
|---|---|---|
| Does this prompt fit the core @gimpysticks identity? | Consider Reel or feed carousel. | Use Stories or skip feed. |
| Does it have one strong visual mechanic? | Build image/Reel. | Rewrite before producing. |
| Can it become a comment question? | Add CTA and post. | Use as private archive or Story. |
| Can it be saved or shared? | Build carousel/Reel packaging. | Do not boost. |
| Is it one of multiple posts today? | Delay or convert to Story. | Publish only if it has a clear job. |

## Suggested operating model

For each day, the social management process should be:

1. Select the daily prompt from the appropriate challenge list.
2. Load the correct style system for that challenge.
3. Decide whether the output belongs in Feed, Reel, Stories, or archive-only.
4. Generate a Midjourney prompt using the correct output order.
5. Rewrite the caption for Instagram engagement, not just lore.
6. Add a Reel adaptation if the image has enough visual clarity.
7. Add a Story poll or question that extends the post.
8. Publish only one major post per day unless there is a strategic reason to do otherwise.
9. Track reach, saves, shares, comments, and follows after 24–48 hours.

## Bottom-line interpretation

Your files confirm that you already have a sophisticated creative production system. What needs to change is **distribution discipline**. The template system should not automatically create more posts. It should create better candidates, and then we should choose which candidates deserve feed/Reel treatment.

The best way forward is to convert these templates into a managed content pipeline: **Challenge Prompt → Style System → Prompt Output → Caption Rewrite → Reel/Story Packaging → Approval → Publish → Insight Review**.
